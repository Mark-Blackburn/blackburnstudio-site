# Evidence Audit Pack

This folder contains internal factual audits for candidate Digital projects.

It also contains an internal project-highlight planning layer under [project-highlights/](project-highlights/README.md), used to decide launch roles, safe claims, asset needs, and case-study readiness.

Method rules used in this pack:

- Evidence is limited to repositories and documentation available in this workspace, plus sources explicitly linked from those documents.
- Where a relevant repository is not available, technical evidence is marked: Requires repository review.
- Where facts cannot be established from source or documentation, they are marked: Requires owner input.
- Implemented and verified work is separated from documented-only or planned work.
- No public case-study copy is drafted here.
