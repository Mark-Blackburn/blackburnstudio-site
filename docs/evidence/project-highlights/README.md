# Project Highlight Outlines

These files are internal planning outlines for deciding which projects support the Digital launch, how they should be labelled, what they prove, and what evidence or assets are still needed before any public use.

## Purpose

Use these outlines to support decisions about:

- which projects should appear on the `/digital` launch page
- which projects may appear briefly on the homepage
- which projects could later become fuller case studies
- what each project proves
- which services each project supports
- which claims are safe
- which visual assets are needed
- how commercial, internal, and volunteer work should be labelled

## Document types

### Evidence file

An evidence file records the factual basis for a project, including contribution boundaries, permissions, outcomes, evidence strength, and open questions.

### Project-highlight outline

A project-highlight outline is an internal planning layer built from the evidence file. It helps decide launch role, safe claims, visual needs, and whether the project is ready for brief public reference.

### Public case study

A public case study is a later editorial deliverable. It requires stronger evidence control, approved assets, approved wording, and a deliberate decision about how the project should be presented externally.

Do not treat these outline files as public copy.

## Evidence and permission rules

- Follow the governing evidence files and the candidate project register as the source of truth.
- Do not convert planned work into delivered work.
- Do not introduce unsupported metrics, testimonials, or quotations.
- Keep commercial, volunteer, and internal work visibly distinguished.
- Do not imply endorsement from AFL, PlayHQ, Microsoft, Meta, or other third parties.
- Do not expose private operational information, board material, member data, administrator data, or security-sensitive material.
- Use only approved visuals or visuals marked as requiring anonymisation or recreation.
